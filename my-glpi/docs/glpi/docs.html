GLPI 11.0.7 - Règles Métier

1. Gestion des Tickets

Types de tickets

  • Incident : dysfonctionnement ou panne.
  • Demande : besoin ou requête utilisateur.

Cycle de vie d'un ticket

Statut Description
New Ticket nouvellement créé.
Approval En attente de validation.
Processing (Assigned) Assigné à un technicien, groupe ou fournisseur.
Processing (Planned) Une tâche planifiée existe.
Pending En attente d'informations ou d'action externe.
Solved Solution proposée.
Closed Solution validée et ticket clôturé.

Règles automatiques

  • Tout ticket commence à l'état New.
  • Une assignation passe le ticket en Processing (Assigned).
  • Une tâche planifiée passe le ticket en Processing (Planned).
  • L'ajout d'une solution passe le ticket en Solved.
  • La validation de la solution passe le ticket en Closed.

Priorité

Priorité = Matrice(Urgence, Impact)
  • L'utilisateur définit l'urgence.
  • Le technicien définit l'impact.
  • GLPI calcule automatiquement la priorité.

Règles Métier (Business Rules)

Les règles métier permettent de modifier automatiquement un ticket lors de sa création ou de sa mise à jour.

Exemples :
  • Catégoriser automatiquement un ticket.
  • Assigner automatiquement un groupe.
  • Assigner automatiquement un technicien.
  • Modifier l'urgence.
  • Modifier la priorité.
  • Lancer une validation.
  • Associer automatiquement un équipement.

2. Gestion des Assets

Un Asset représente un élément du parc informatique ou matériel.

Types d'Assets disponibles

  • Ordinateurs
  • Moniteurs
  • Imprimantes
  • Téléphones
  • Équipements réseau
  • Périphériques
  • Logiciels
  • Cartouches
  • Consommables
  • Racks
  • PDU
  • Câbles
  • Cartes SIM
  • Assets personnalisés (GLPI 11)

Relations entre Assets

  • Un ordinateur peut posséder plusieurs composants.
  • Un écran peut être lié à un ordinateur.
  • Un périphérique peut être partagé.
  • Un utilisateur peut être affecté à plusieurs assets.
  • Un asset appartient à une entité.

Informations importantes

  • Nom.
  • Numéro d'inventaire.
  • Numéro de série.
  • État.
  • Localisation.
  • Utilisateur affecté.
  • Informations financières.

3. Gestion de l'Inventory

Sources d'inventaire

  • Création manuelle.
  • GLPI Agent.
  • Native Inventory.
  • SNMP Inventory.

Cycle d'inventaire

  1. L'agent collecte les informations.
  2. Les données sont envoyées à GLPI.
  3. GLPI applique les règles d'inventaire.
  4. Les assets sont créés ou mis à jour.
  5. Les composants sont synchronisés.

Règles d'import

  • Identification via numéro de série.
  • Identification via UUID.
  • Identification via MAC Address.
  • Mise à jour automatique si l'asset existe déjà.
  • Création automatique sinon.

Règles de découverte réseau

  • Découverte des équipements sur le réseau.
  • Classification automatique par type.
  • Import SNMP des informations techniques.

Assets personnalisés (GLPI 11)

  • Création de nouveaux types d'assets.
  • Champs personnalisés.
  • Intégration avec l'inventaire natif.
  • Gestion via API REST v2.

4. Relations entre les modules

Module Relation
Ticket → Asset Un ticket peut être lié à un ou plusieurs assets.
Inventory → Asset L'inventaire crée ou met à jour les assets.
Ticket → Utilisateur Un utilisateur est demandeur ou technicien.
Asset → Utilisateur Un asset peut être affecté à un utilisateur.
Asset → Entité Chaque asset appartient à une entité GLPI.
